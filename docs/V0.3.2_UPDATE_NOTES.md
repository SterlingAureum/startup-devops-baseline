## Why values.yaml is not changed by default

The current repository still supports local kind development. The default `values.yaml` can remain local-friendly:

```yaml
image:
  repository: startup-devops-baseline/demo-api
  pullPolicy: Never
```

Use `scripts/set-demo-api-image.sh` to explicitly switch to a GHCR image tag when you want to test the registry-based release flow.

## Suggested apply steps

```bash
unzip startup-devops-baseline-v0.3.2-ghcr-image-publishing-update.zip
cp -r startup-devops-baseline-v0.3.2-ghcr-image-publishing-update/* .
cp -r startup-devops-baseline-v0.3.2-ghcr-image-publishing-update/.github .
chmod +x scripts/set-demo-api-image.sh scripts/set-demo-api-local-image.sh scripts/check-ghcr-demo-api-image.sh
```

Commit:

```bash
git add .github apps/demo-api/helm/values-ghcr.yaml scripts docs/GHCR_IMAGE_WORKFLOW.md docs/V0.3.2_UPDATE_NOTES.md
git commit -m "feat: add ghcr image publishing workflow"
git push
```

After GitHub Actions publishes an image, deploy it with:

```bash
IMAGE_TAG=sha-<short-commit> ./scripts/set-demo-api-image.sh
git add apps/demo-api/helm/values.yaml
git commit -m "release: update demo-api image to sha-<short-commit>"
git push
```

Then watch rollout:

```bash
./scripts/rollout-watch.sh
```
