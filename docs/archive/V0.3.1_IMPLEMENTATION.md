# v0.3.1 Ingress Canary Routing Implementation

This update implements ingress-based canary routing for `demo-api` using Argo Rollouts and ingress-nginx.

## What changes

Before v0.3.1, `demo-api` used a Rollout but ingress still pointed to a single service. The canary behavior mainly demonstrated Rollout state and ReplicaSet transitions.

In v0.3.1, the chart renders:

```text
rollout/demo-api
service/demo-api-stable
service/demo-api-canary
ingress/demo-api
```

The stable ingress points to `demo-api-stable`. Argo Rollouts controls NGINX canary routing by referencing the stable ingress in the Rollout strategy.

## Resource relationship

```text
Client
  |
  v
ingress-nginx
  |
  +--> service/demo-api-stable
  |
  +--> service/demo-api-canary
          ^
          |
    Argo Rollouts adjusts canary weight
```

## Important Argo CD note

Argo Rollouts updates the selectors of the stable and canary Services during rollout. The `demo-api` Argo CD Application therefore includes `ignoreDifferences` for the Service selectors and `RespectIgnoreDifferences=true`.

Without this, Argo CD self-heal may try to revert selector changes made by Argo Rollouts.

## Validate locally

```bash
helm lint apps/demo-api/helm
helm template demo-api apps/demo-api/helm | grep -E "kind: Rollout|kind: Service|kind: Ingress|demo-api-stable|demo-api-canary|stableIngress"
```

After Argo CD sync:

```bash
kubectl get application -n argocd
kubectl get rollout -n startup-apps
kubectl get svc -n startup-apps
kubectl get ingress -n startup-apps
./scripts/validate.sh
```

## Trigger a canary test

Build and load a new local image tag:

```bash
docker build -t startup-devops-baseline/demo-api:0.1.1 apps/demo-api
kind load docker-image startup-devops-baseline/demo-api:0.1.1 --name startup-devops-baseline
```

Then update:

```yaml
image:
  tag: "0.1.1"

env:
  APP_VERSION: "0.1.1"
```

Commit, push, and watch:

```bash
./scripts/rollout-watch.sh
```

## Acceptance criteria

- `rollout/demo-api` is Healthy.
- `service/demo-api-stable` exists.
- `service/demo-api-canary` exists.
- `ingress/demo-api` exists and points to `demo-api-stable`.
- Rollout strategy includes `trafficRouting.nginx.stableIngress: demo-api`.
- `./scripts/validate.sh` passes.
