# v0.3.3 AnalysisTemplate Helm Fix

## Problem

`helm lint apps/demo-api/helm` failed with:

```text
error calling dig: interface conversion: interface {} is chartutil.Values, not map[string]interface {}
```

The original template used:

```gotemplate
{{- if (dig "analysis" "enabled" false .Values) }}
```

In this chart context, Helm passed `.Values` as `chartutil.Values`, which caused `dig` to fail.

## Fix

The template now uses Helm-compatible `default dict` variables:

```gotemplate
{{- $analysis := .Values.analysis | default dict }}
{{- if ($analysis.enabled | default false) }}
```

This avoids the `dig` type conversion issue and keeps the template safe when `analysis` is not defined.

## Apply

```bash
helm lint apps/demo-api/helm
helm template demo-api apps/demo-api/helm | grep -E "kind: AnalysisTemplate|demo-api-canary-health" -A20
```
