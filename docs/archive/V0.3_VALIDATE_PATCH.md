# v0.3 Validate Script Patch

This patch updates `scripts/validate.sh` for the v0.3 Progressive Delivery baseline.

## Why this patch is needed

Before v0.3, `demo-api` was deployed as a standard Kubernetes Deployment.

In v0.3, `demo-api` is managed by Argo Rollouts as a `Rollout` resource. The old validation script still checked:

```bash
kubectl rollout status deployment/demo-api -n startup-apps
```

That caused a false failure:

```text
FAIL: demo-api deployment not found: demo-api
```

The application may still be healthy because the pods are now created through a Rollout-managed ReplicaSet.

## What changed

The new script now:

- checks the Argo Rollouts Application if it exists
- checks the `argo-rollouts` controller if installed
- checks `rollout/demo-api` first
- falls back to `deployment/demo-api` for older versions
- uses `kubectl argo rollouts status` when the plugin is available
- uses a kubectl polling fallback when the plugin is unavailable

## Helper script naming update

This patch also clarifies rollout helper scripts:

```text
scripts/rollout-status.sh  # one-shot status output
scripts/rollout-watch.sh   # continuous watch output
```

This avoids confusion where a `status` script appears to run forever.
