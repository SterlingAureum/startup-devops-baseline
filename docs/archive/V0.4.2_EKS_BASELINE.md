# v0.4.2 EKS baseline

## Goal

Create the first AWS-managed Kubernetes runtime for the repository while
keeping application delivery, Karpenter, and ingress integration outside this
phase.

## Infrastructure

- EKS control plane across the existing private subnets;
- one EKS managed node group in private subnets;
- On-Demand `t3.medium` nodes by default;
- cluster and node IAM roles;
- OIDC provider and EBS CSI IRSA role;
- EKS managed VPC CNI, kube-proxy, CoreDNS, and EBS CSI add-ons;
- optional administrator access entry.

## Security decisions

- worker nodes do not receive public IP addresses;
- the Kubernetes API supports private VPC access;
- public endpoint access remains enabled for local bootstrap;
- public API CIDRs must be narrowed before leaving the environment online;
- workload AWS permissions are separated from the node role where implemented;
- Terraform does not store AWS credentials.

## Deferred work

- AWS Load Balancer Controller;
- Argo CD bootstrap;
- application deployment;
- Karpenter and Spot capacity;
- remote Terraform state;
- full control-plane logging and production hardening.

## Validation

```bash
./scripts/validate-terraform.sh

terraform -chdir=infra/terraform/aws/environments/dev plan
terraform -chdir=infra/terraform/aws/environments/dev apply

./scripts/validate-eks-baseline.sh
```

Destroy the environment when it is not actively being used because EKS,
managed nodes, NAT Gateway, and EBS volumes incur ongoing charges.
