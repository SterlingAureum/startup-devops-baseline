# v0.5.0 Karpenter AWS Foundation

## Purpose

This increment prepares the AWS resources required by Karpenter before the
controller and its Kubernetes custom resources are installed through Argo CD.

## Delivered

- EKS 1.36 pinned for repeatable Karpenter compatibility
- stable two-node On-Demand Managed Node Group labeled `workload=system`
- dedicated Karpenter node IAM role
- EKS `EC2_LINUX` access entry for Karpenter nodes
- Karpenter controller IRSA role
- six scoped Karpenter 1.14 controller IAM policies
- encrypted SQS interruption queue
- EventBridge rules for health, Spot, rebalance, instance state, and capacity reservation events
- Karpenter discovery tags on private subnets and the EKS cluster security group
- Terraform outputs consumed by the next GitOps increment
- AWS foundation validation script

## Ownership Boundary

```text
Terraform
├── IAM roles and policies
├── EKS access entry
├── SQS interruption queue
├── EventBridge rules and targets
└── subnet and security-group discovery tags

Argo CD - deferred to v0.5.1
├── Karpenter CRDs
├── Karpenter controller
├── EC2NodeClass
└── NodePool
```

Karpenter does not provision EC2 nodes in v0.5.0 because no controller,
`EC2NodeClass`, or `NodePool` exists yet.

## Apply

If `terraform.tfvars` already exists from v0.4, update it explicitly before
planning:

```hcl
eks_cluster_version = "1.36"
eks_node_min_size    = 2
```

```bash
terraform -chdir=infra/terraform/aws/environments/dev init
terraform -chdir=infra/terraform/aws/environments/dev plan -out=tfplan
terraform -chdir=infra/terraform/aws/environments/dev show tfplan
terraform -chdir=infra/terraform/aws/environments/dev apply tfplan
```

## Validate

```bash
./scripts/validate-karpenter-foundation.sh
```

Expected results:

- six managed policies attached to the Karpenter controller role
- four AWS managed policies attached to the Karpenter node role
- Karpenter node access entry present in EKS
- encrypted interruption queue present
- five EventBridge interruption rules present
- at least two private subnets have the discovery tag
- the EKS cluster security group has the discovery tag
