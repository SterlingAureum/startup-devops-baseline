# v0.5.5 Karpenter AWS FIS Spot Interruption

## Scope

This increment adds a real, controlled Spot interruption drill:

- Terraform-managed AWS FIS IAM and experiment-template resources;
- a test-only EC2NodeClass with a unique EC2 target tag;
- a test-only Spot NodePool with a distinct scheduling taint;
- zero-fault validation of the FIS and Kubernetes contracts;
- an explicitly confirmed drill that starts AWS FIS and verifies Karpenter
  recovery.

## Blast-radius Contract

The experiment target is:

```text
resource type: aws:ec2:spot-instance
target tag:    KarpenterFISTest=startup-devops-baseline-dev-spot-interruption
state filter:  running
selection:     COUNT(1)
```

Only instances launched through `EC2NodeClass/application-fis` receive that
tag. The normal `application-spot` pool continues to use the shared
`EC2NodeClass/application` and is not eligible for the experiment.

Immediately before starting FIS, the drill queries EC2 with the same tag and
requires exactly one running Spot instance. It also requires that instance to
match the node hosting the temporary test Pod.

## Capacity Contract

The FIS-only NodePool uses:

```yaml
capacity-tier: spot-fis
dedicated=application-spot-fis:NoSchedule
```

It permits only generation 6 or newer Linux `amd64` Spot instances from the
`c`, `m`, and `r` categories with 2 vCPUs. The pool permits at most two nodes,
four total vCPUs, and 32 GiB total memory so Karpenter can create a replacement
while the interrupted node is still draining, including when the first choice
is a memory-optimized instance.

## Apply and Validate the Foundation

After merging or pushing the increment:

```bash
./scripts/validate-terraform.sh
terraform -chdir=infra/terraform/aws/environments/dev plan -out=tfplan
terraform -chdir=infra/terraform/aws/environments/dev show tfplan
terraform -chdir=infra/terraform/aws/environments/dev apply tfplan
```

Allow Argo CD to sync the new EC2NodeClass and NodePool, then run:

```bash
./scripts/validate-all.sh
```

This does not start an FIS experiment or create EC2 capacity.

## Run the Real Interruption Drill

Run intentionally:

```bash
./scripts/run-karpenter-fis-spot-test.sh
```

The script requires typing `interrupt`. For non-interactive automation, the
same acknowledgement can be supplied explicitly:

```bash
CONFIRM_FIS_INTERRUPT=interrupt \
  ./scripts/run-karpenter-fis-spot-test.sh
```

The drill:

1. validates the SQS/EventBridge interruption path and FIS foundation;
2. confirms the dedicated FIS pool is empty;
3. creates one temporary Pod and Spot node;
4. proves the FIS tag resolves to exactly that instance;
5. starts the `aws:ec2:send-spot-instance-interruptions` action with `PT2M`;
6. waits for a new Ready Pod on a different Spot instance;
7. verifies FIS completion, original EC2 termination, and original NodeClaim
   removal;
8. removes the workload and waits for final scale-in.

The action sends a real EC2 interruption notice. Once EC2 accepts the
interruption, stopping the FIS experiment cannot undo it. The drill incurs
temporary EC2, EBS, and AWS FIS charges, and Spot capacity can be unavailable.

Expected final output:

```text
Karpenter AWS FIS Spot interruption and replacement validation passed.
```
